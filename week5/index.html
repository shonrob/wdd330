Testing and Debuggin

  • alert() - stops the program from running until OK is clicked.
  • console.trace() method will log an interactive stack trace to the console. This shows the functions that were called in the lead up to an exception occuring while the code is running.
  • debugger: breakpoints in the code that will pause the execution and allow the programer to see where the program is currently at.
    • hover: over any variable to seew hat value it currently holds.
    • Don't forget to remove the break point so when others go to use the code the code can execute throughly.
  • Error Objects
    • EvalError: used to identify errors when using the global eval() function.
    • RangeError: thrown whena number is outside an allowable range in values.
    • ReferenceError: thrown when a reference is made to an item that doesn't exist.
    • SyntaxError: thrown when there's an error in teh code's syntax.
    • TypeError: thrown when there's an error in the type of value used.
    • URIError: thrown when there's a probelm encoding and decoding the URI
    • InternalError: is a non-standard error that is thrown when an error occurs in the JavaScript engine.--a common cause of this is too much recursion.
    • Since error objects have a series of properties they are often used inconsistently across browsers. To be safe the:
      • name property returns the name of the error constructor function used as a string, such as 'Error' or 'ReferenceError'.
      • message property returns a description of the error and should be provided as a argument to the Error constructor function.
      • stack property will return a stack trace for that error. This is a non-standard property and it's recommended that it is not safe to use in production sites.
