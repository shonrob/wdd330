  • Objects are copied by reference- that means objects are assigned by reference..the object already exists..in the exact same space in memory.

The Document Object Model

  • Document Object Model (DOM) allows access to elements of a web page and enables interaction with the page by adding and removing elements, changing the order of content and attributes of elements and even altering how they are styled.
  • The DOM treats everything on the web page as a node.
  • Use JavaScript to access and modify different parts of a web page using a special built-in object called document
  • Tips for using the console: pressing TAB will autocomplete any methods and should show you a list of a possible methods..Pressing the UP arrow key will select the previous command entered.
  • to reference a body element we can write typeof
  • Legacy DOM Shortcut Methods: Document.body, Document.images, Document.links, Document.anchors, Document.forms
  • Node lists are array like objects but they are not arrays.
  • getElementsById()
  • getElementsByTagName()
  • getElementsByClassName()
  • document.querySelector() : method allows you to use CSS notation to find the first element in the document that matches a CSS selector provided as an argument. If no elements match, it will return null
  • document.querySelectorAll(): methods that gets all elements
  • getAttribute() method returns the value of the attribute provided as an argument
  • setAttribute() can change the value of an elements attributes. It takes 2 arguments: the attribute that you wish to change and the new value of the attribute.
  • Creating an Element: createElement() method that takes a tag name as a parameter and returns that element.
  • Creating a Text Node: document.createTextNode() method takes a parameter, which is a string containing the text that goes in the node.
  • Then after creating a text node we need to append the node: appendChild() method that will add another node (given as argument) as a child node.
  • So we Create the element node, then create the text node, then append the text node to the element node
  • Simpler to use textContent : example: const flash = document.createElement('li'); flash.textContent = 'Flash';
  • A Function to Create Elements: When we created our new list item element, all we specified was the type of tag and the text inside it. These will form the parameters of our function. The function will then perform the two steps we used to create the new element, and then return that element
  • we can remove or replace child : removeChild() or replaceChild()
  • innerHTML: returns all the child elements of an element as a string of HTML. If an element contains lots of other elements, all the raw HTML is returned.

Chapter 7 Events

  • Events are another part of the DOM that provides the link between the web page and the user interactions
  • Event Listeners: A way for js to alert when something is happening: document.body.addEventListener("click", doSomething);
  • click Event: click event occurs when a user clicks a mouse, presses the Enter key, or taps the screen.
  • only one event handler for each event-type can be attached to an element
  • the code for the even handlers is hidden away in the mark up, making it difficult to find where these events have been delcared.
  • the js code has to be entered in a string, becareful when using apostrophes and quote marks.
  • The first parameter is the type of event, the second is a call back function that is invoked when the event occurs.
  • The Event Object: whenever an even handler is triggered by an event, the callback function is called. The function is automatically passed an event object as a parameter that contains info about the event.
  • function doSomething(event){console.log(event.type);
  • The parameter does not have to be called event. it can be any legal variable name, although calling it event can make it easier to read the code. many developers often abbreviate it to just e
  • Event Target: target property returns a reference to the node that fired the event. If you change the doSomething() function it will show a message in the console telling us the node that was clicked on.
  • Cooridnates of an Event: The screenX and screenY properties show the number of pixels from the left and top of the screen respectively where the event took place....The clientX and clientY properties show the number of pixels from the left and top of the client that is being used (usually the browser window)......The pageX and pageY properties show the number of pixels from the left and top, respectively, where the event took place in thedocument. This property takes account of whether the page has been scrolled.
