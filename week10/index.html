MDN: Validating Forms

    Client-side for validation: helps ensure data submitted matches the requirements set forth in the various form controls.
  • Security checks need to be performed on the server-side and the client side
  • We want to get the right data, in the right format.
  • We want to protect our users' data
  • We want to protect oursevles.
      • Different Types of Client-side validation
    • Built-in form validation: better than JavaScript but not as customizable.
      • required
      • minlength/maxlength
      • min and max
      • type: number, email, etc
      • pattern: regex
    • JavaScript: Completely customizable, but you have to create it all or use a library.
        • The Constraint Validation API
      • Elements to use: button, fieldSet, input, output, select, textArea
      • properties for the elementsvalidationMessage, validity-returns a ValidityState object: patternMismatch, tooLong, maxlength, tooShort, minlength, rangeOverflow, rangeUnderflow, typeMismatch, valid, valueMissing, willValidate
      • methods checkValidity(), setCustomValidity

MDN: Using Fetch

    fetch()
  • takes one argument- the path to the resource you want to fetch and returns a promise containing the response (a response object)
  • can optionally accept a second parameter, an init object that allows you to control a number of different settings.
  • Sending a request with credentials included: both the same-origin and a cross-orgin calls, add credentials: 'include' to the 'init' object you passed in the fetch() method..if the orgin is the same as the request URL: credentials:'same-origin'. Or you can 'omit' the credentials all together.
  • files can be uploaded using an HTML input type = 'file' and formData(). Also, multiple files can be used.
  • Body: ArrayBuffer, ArrayBufferView, Blob, string, URLSearchParams, FormData
