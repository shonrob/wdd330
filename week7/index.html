Chapter 11-Furthering Functions

      Call and Apply Methods
    • call() method can be used to set the value of this inside a function to an object that is provided as the first argument.
    • Create some objects that a property, then us the call() method to invoke a function, providing that each object as an argument. It will take the value of this in the function.
    • The apply() method works in the same way, except the arguments of the function are provided as an array, even if there is only one argurment.
      Immediately Invoked Function Expressions
    • An Immediately Invoked Function Express is an anonymous function that, as the name suggest, is invoked as soon as it's defined.
    • These are a useful way of performing a task while keeping any variables wrapped up within the scope of the function. So the global namespace is not polluted with lots of variable names.
      Temporary variables
    • You cannot remove a variable from a scope once it has been declared.
      Initialization Code
    • An immediately invoked function expression can be used to set up any initialization code that there'll be no need for again.
    • The code can only be run once. There is no need to to create any reusable, name function, and all the variables will also be temporary.
      Functions that Define and Rewrite themselves
    • Because JavaScript is dynamic it can call itself, define itself, and redefine itself. It is done through assigning an anonymous function. To a variable that has the same name as the function.
    • If properties have been defined on the first set of the function, those will be lost wehn the function redefines it self.
      Recursive Functions
    • A recursive Funtion is one that invokes itself until a certain condition is met.
    • It's a useful tool to use when iterative processess are involved.
    • A promise represents the future result of an asynchronous operation. Promises don't do anything different than a callback but they simplify the process and avoid the convoluted code that could cause multiple callbacks.
    • When a promise is created it calls an asynchronous operation that is said to be pending. It stays in that state while the operation is taking place. This is known as to be unsettled Once the operation has completed, the promise is settled
    • Resolvedthe ansynchronous operation was completed successfully.
    • Rejectedthe ansynchronous operation didn't work as expected, wasn't succesfully completed or resulted in an error.
    • A promise is created using a constructor function. This takes a function called an executor as an argument. The executor initializes the promise and starts the asynchronous operation. It accepts two functions as arguments: the resolve() function if function is successful. And the reject() function is called if the operations fail.
        • Dealing with a Settled Promises
      • then()method can be used to deal with the outcome. The method accepts 2 arguments. The first is a fulfilment function that's called when the promise is resolved. Any data returned from the resolve() function will be passed along this function. The second argument is a rejection function that's called if the promise is rejected.
      • Alternatively, the catch method can be used to specify what to do if the operations fails instead.
      • The then()and the catch() methods can be chained together to form a succinct description of how to deal with the outcome of the promise.
      Functional Programing
    • JavaScript has always supported function-style programming because the functions are being first-class objects.
    • The ability to pass functions as arguments, return them from other functions, and use anonymous functions is what allows JS to excel.
    • JavaScript is a multi-paradigm language.
    • A pure function is a function that adheres to the following rules.
      1. The return value of a pure function should only depend on the values provided as arguments. It doesn't rely on values from somewhere else in the program.
      2. There aren't side effects. A pure function doesn't change any values or data elsewhere in the program. It only makes non-destructive data transformations and returns new values, rather than altering any of the underlying data.
      3. Referential transparency. Given the same argurments, a pure function will always return the same result.
          4. A Pure Functions must have
        • At least one argument; otherwise the return value must depend on something other than the arguments of the function, break the firsst rule
        • A return value; otherwise there's no point in the function (unless it has changed somehting else in the program- in which case, it broken the 'no side-effects' rule).
    • Functional programing uses pure functions as the building blocks of a program.
      Higher-Order Functions
    • High-order functions are functions that accept another function as an argument, or return another funtion as a result, or both.
    • Closures are used extensively in higher-order functions as they allow us to create a generic function that can be used to then return more specific functions based on its arguments. This is done by creating a closure around a function's arguments that keep them 'alive'.

AJAX

  • Ajax is a technique that allows web pages to communicate asynchronously with a server, and it dynamically updates web pages without reloading. Enabling the data to be sent and received in the background, as well as portions of the page to be updated in response to user events, while the rest of the program continues to run.
  • Typically a server is needed to run Ajax. A server side language like PHP, Ruby, Node.js or .NET help serve the data response following an Ajax request.
