Think, Design, Test, Create

I appreciate how "Doing more with web stuff" starts with the steps to

  • Think
  • Design
  • Test
  • Create

because it helps break down the code. I often have the tendency to see the entire project and get overwhelmed. However, with these steps I can break the project into parts and think through each portion. I know I will have many redo projects and repeating the steps, but I look forward to piecing the page together.

I love the simplicity of the book as it walks through all the information for anyone. It would have been a great resource when I started my journed to learning about coding. I started knowing nothing and with each course I take I have the opportunity to learn more even though I don't master it.

Master Mobile UX Notes

  • Important to design mobile first
  • Helps to make the web more user friendly so you don't have to redesign.
  • Start small, scale up.
  • Be There: Need to know and meet their expectations
  • Be Useful: Simple, but eye catching design. User wants to easily access the site.
  • Be Quick: Attention span of current kids is 8 seconds...if they can't find what they need they leave.
  • Think about the device and make it easier for user to quickly click without moving hands to much.
  • Optimize files-so user's devices can load faster
  • Empathtic to everyones needs
  • Simple, Fluid, Reachable
  • Menu button is more prominently used compared to the hamburger button.
  • Promote loyalty
  • Only accept the most important info- simplify
  • Progress bar, autofill-simple, easy, and quick, auto spacing
  • Number pad
  • Serving end users
  • No pop ups!! Annoying
