Chapter 8:CSS3 Transforms and Transitions

    Transforms
  • transform property allows you to translate, rotate, scale, and/or skey any element on the page.
  • Transforms don't work on inline elements. So be sure to display: inline-block;
  • If you rotate before translating, your translate direction will be on the rotated axis.
    Translation
  • translate functions allow you to move elements left, right, up, or down.
  • When using translation function, the element moves without impacting the flow of the document
  • Unlike position: relative which allows you to position an element either against its current position or against a parent or other ancestor, a translated element can only be moved relative to its current position.
  • translate(x,y) function moves an element x from left and y from the top.
  • With translate you don't have to use both x and y you can use one or the other or both.
    Scaling
  • scale(x,y) function scales an element by the defined factors horizontally then vertically.
  • If only one value is provided it will be used for both the x and y values. Growing/shrinking your elements while maintaining the original aspect ratio.
  • ie: scale(1) would leave the element the same, scale(2) would double it proportions, scale(0.5) would halve them. if you provide different values for the x and y it will distort the element.
  • By default the point of origin is the elements center
    Rotation
  • rotate() function rotates an element around the point of origin by a specified angle value.
  • Angles are declared in degrees. Positive degrees move clockwise...Negative move counterclockwise.
  • Technically, you can use the values in grads, radians, or turns not just degrees.
    Skew
  • skew(x,y) function specifies a skew along the x and y axes. x specifies on the x axis. y specifies on the y axis. If the second parameter is omitted, the skew will only occur on the x axis.
  • There are axis-specific versions of the skew transform: skewX() and skewY()
    Transitions
  • Transitions allow the values of CSS properties to change over time.
      • Steps to Create a simple transition using only CSS:
    1. Declare the original state of the element inthe default style declaration.
    2. Declare the final state of the tranisitioned element..ie :hover state
    3. Include the transition functions in your default style declarations using the transition p
      transition-property
    • defines the CSS properties of the element that should be transitioned, with all for all properties being the default.
      transition-duration
    • sets how long the transition will take: the duration of time it takes to go from the default state to the transitioned state.
    • It can be specified in seconds(s) or milliseconds(ms)
    • Ideal timing would be (0.2s) or (200ms)
    • can be listed as 4 different properties or all four can be establish in one line of code: element {transform 0.2s ease-out 50ms; transition-property; transition-duration; transition-timing-function; transition-delay;}
    • Multiple Transitions on an element: ie: transition: transform 0.2s ease-out 50ms, color 0.1s linear 50ms;
      transition-timing-function
    • allow to control the pace of the transtion for a more granular detail.
        • most common timing functions include the key terms:
      • ease: has a slow start then it speeds up, and ends slowly
      • linear: creates a transition that animates at a constant speed
      • ease-in: creates a transition that is slow to start but gains speed, then stops abruptly
      • ease-out: starts at full speed, then slows progressively as it reaches the conclusion of the transition
      • ease-in-out: accelerates more sharply at the beginning
      • cubic-bezier(0, 0, 1, 1)

      Transition-delay
    • Possible to start the delay before the transition begins
    • Better UI if there is a 50ms delay for action to happen to be sure the user purposely is hovering.
    • When using the all keyword, all the properties transition at the same speed, rate, and delay. ie: transition: all 0.2s ease-out 50ms;
